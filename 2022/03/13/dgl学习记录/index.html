1.2 图、节点和边

如果想创建一个上面这样的图，需要创建两个表示边的tensor，然后调用dgl.graph()方法。这两个tensor分别是源节点和目标节点的ID：参考官方文档

import torch
import dgl
u = torch.tensor([0,0,0,1])
v = torch.tensor([1,2,3,3])
g = dgl.graph((u,v))

1.3 节点和边的特征

可以通过DGLGraph的ndata和edata接口访问图的具名特征：（假设图g已经创建）

g.ndata["x"]= torch.randn(g.num_nodes(),10)
g.edata["x"] = torch.rand(g.num_edges(),5)

1.5 异构图

若一个图中，有不同类型的边和节点，那么这个图就是异构图。在dgl中，用DGLHeteroGraph表示一个异构图。用dgl.heterograph()方法可以创建异构图，其第一个参数是一个字典。字典的键是元组，用以表示节点类型-边类型-节点类型的连接类型；字典的值是表示两个边的tensor（和同构图一样）。如下：

import torch
import dgl

u = torch.tensor([0, 0, 0, 1])
v = torch.tensor([1, 2, 3, 3])
g = dgl.heterograph({
    ("drug", "interacts", "drug"): (u, v),
    ("gene", "interacts", "drug"): (u, v)
})
print(type(g))
# <class 'dgl.heterograph.DGLHeteroGraph'>
print(g)
# Graph(num_nodes={'drug': 4, 'gene': 2},
#       num_edges={('drug', 'interacts', 'drug'): 4, ('gene', 'interacts', 'drug'): 4},
#       metagraph=[('drug', 'drug', 'interacts'), ('gene', 'drug', 'interacts')])

第2章：消息传递范式

2.1 内置函数和消息传递API

2.1.1 消息的生成

message是边的特征。即，在图的edata属性中。根据据源节点、目标节点和边这三者的特征可以计算出message。比如，下面这个函数指定计算g.edata["he"]的方式，具体为：

g.edata["he"] =源节点的x特征+目标节点的y特征：

def message_fn(edges):
    return {"he": edges.src["x"] + edges.dst["x"]}

这个函数的参数edges有三个属性src,dst,data，分别用来访问源节点、目标节点和边的特征 。定义好传播函数后，就可以在图上传播了：

g.apply_edges(message_fn)

完整代码：

import dgl
import torch

u = torch.tensor([0, 0, 0, 1])
v = torch.tensor([1, 2, 3, 3])
g = dgl.graph((u, v))
g.ndata["x"] = torch.ones(g.num_nodes(),3)
g.ndata["y"] = torch.ones(g.num_nodes(),3) + 1

def message_fn(edges):
    return {"he": edges.src["x"] + edges.dst["y"]}


g.apply_edges(message_fn)
print(g.edata)
# {'he': tensor([[3., 3., 3.],
#         [3., 3., 3.],
#         [3., 3., 3.],
#         [3., 3., 3.]])}

2.1.2 内置函数

类似上面的message_fn()，框架已经提供了很多消息函数和聚合函数。

比如：将源节点（u）的特征fn与边特征a相乘，得到消息m：

import dgl.function as fn
fn.u_mul_e('ft', 'a', 'm')

然后对所有m消息求和，得到新的节点特征ft：

fn.sum('m', 'ft')

完整代码：

def update_all_example(graph):
    # 在graph.ndata['ft']中存储结果
    graph.update_all(fn.u_mul_e('ft', 'a', 'm'),
                     fn.sum('m', 'ft'))
    # 在update_all外调用更新函数
    final_ft = graph.ndata['ft'] * 2
    return final_ft

作用：将多个图放到一起，组成一个非连通图。

如：

import dgl
from networkx import draw

2.带有feature的batch

当每个图都有相同名字的特征时（且维度也相同），batch生成的图也有特征，其特征是单个图特征的拼接：

import torch
import dgl

g1 = dgl.rand_graph(10, 50)
g2 = dgl.rand_graph(10, 50)
g1.ndata["x"] = torch.randn(10, 3)
g2.ndata["x"] = torch.randn(10, 3)
batch = dgl.batch([g1, g2])
print(batch.ndata['x'])

3.AvgPooling

当一个图被池化，那么输出的是一个(1,N)的向量，表示这个图。

若这个图是batch生成的，那么这个batch里面有几个图，就生成几个图的向量。即：输出(n_batch,N)形状的Tensor。

下面例子中，batch中有两个图。每个节点的x特征维度为3。因此池化输出的结果是(2,3)的Tensor：

forward方法link
import torch
import dgl
from dgl.nn.pytorch import AvgPooling

g1 = dgl.rand_graph(10, 50)
g2 = dgl.rand_graph(10, 50)
g1.ndata["x"] = torch.randn(10, 3)
g2.ndata["x"] = torch.randn(10, 3)
batch = dgl.batch([g1, g2])
print(AvgPooling()(batch, batch.ndata["x"]))

# tensor([[ 0.4438,  0.4382, -0.3256],
#         [-0.2316,  0.1083, -0.1091]])

4.save_graphs

可以将图持久化到磁盘：

graphs = []
labels = []
graph_sizes = []
for i in range(num_graphs):
    graph = dgl.rand_graph(torch.randint(10, 20, size=(1,)).item(), 50)
    graphs.append(graph)
    labels.append(i % 5)
    graph_size = graph.number_of_nodes()
    graph_sizes.append(graph_size)
labels = {
    "graph_labels": torch.LongTensor(labels),
    "graph_sizes": torch.LongTensor(graph_sizes)
}
dgl.save_graphs("small.bin", graphs, labels=labels)

存储图的同时，还存储了labels，这个labels中包含与保存的图相关的标签信息。比如上面这段代码，保存了图的标签（键是graph_labels）和图的大小（键是graph_sizes）。这些数据又能被dgl.load_labels()单独读取出来：

filename="small.bin"
graph_sizes = load_labels(filename)["graph_sizes"]

5.卷积层

GINConv()演示：

即，输入是一个图和其特征，输出是经过一层卷积后的节点的特征。参见文档。

input_dim = 3
n = 5
g = dgl.rand_graph(n, 20)
h = torch.ones(n, input_dim)
print(GINConv()(g,h))

6.TUDataset

这是在“图预测”数据集里的。这个类的实例的__getitem__方法返回一个元组：(dgl.Graph, Tensor)，其中前者是图，后者是图对应的标签。

dataset = TUDataset("enzymes",raw_dir="data")
for g,label in dataset:
    print(g)
    print(label)
    break
